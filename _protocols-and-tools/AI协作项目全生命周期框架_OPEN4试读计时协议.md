# OPEN-4 试读计时协议（最小版）

> **用途**：测"一个没参与设计的人，读懂这套框架到能上手，要多久 + 在哪卡住"。
> **关键纪律**：判据必须在**开测前冻结**（kill-test-first 预登记精神），不许事后调阈值救场。
> **配套**：`AI协作项目全生命周期框架_OPEN4试读计时协议.json`

## 0. 一句话

非设计者冷读框架 → 计时到"能用" + 记误解点 → 对照**预冻结**阈值给 PASS / 改进 / 重做。

## 1. 测试者（谁）

- **必须**：没参与框架设计、没读过审查链、没听过设计者口头解释。
- **画像**：贴近目标用户——个人做项目、有基本 AI 协作经验、但没用过本框架。
- **明确排除**：设计者本人；读过审查链的人；以及 **AI（Opus/DeepSeek 等已带设计上下文）——AI 不能替代此测试，它测的是人类上手成本**。

**测试者经验分档**（v1.6.3 修订）：

| 档位 | 条件 | 优先级 | 说明 |
|------|------|--------|------|
| **A 档** | 有 AI 协作经验（使用过至少一种 AI 编码/协作工具做过项目）+ 未读过框架 | **必测** | 框架目标受众的最小代表——覆盖"有经验但没被作者思路污染"的人 |
| **B 档** | 无 AI 协作经验的新手 | **可选**（若框架公开发布则升级为必测） | 测框架对零经验者的可读性——当前框架不以此为主要目标但需知道边界 |
| **C 档** | 有 AI 协作经验 + 未读过框架 + 非量化/非学术背景 | 可选（扩展档） | 测"非同构读者"的理解障碍——验证 §1.8 局限 #9（作者-读者同构假设）的实际严重性 |

**经验前置声明**：测试者在开测前须声明自己的 AI 协作经验水平——使用过的 AI 工具、做过什么类型的项目、使用时长/频率。声明内容记入测试记录，不做筛选（不是门槛），仅供事后分析"理解障碍是否与经验水平相关"。

## 2. 给什么（材料）

- 框架主文档（`.md` 或 `.docx`），**完整版**。
- **不给**：审查报告、本协议的判据、任何口头导读。
- 任务设定为现实目标：「读到你**能照着最低强制版（§1.4）起一个新项目**」——不要求吃透完整版。

## 3. 怎么测（步骤）

1. 开测前，主持人**先把判据（§5）写下来冻结**，不告诉测试者。
2. 测试者开始读，**计时开始**。允许边读边记笔记 / 出声思考。
3. 主持人**全程不解释、不补充**（一解释就污染计时）——只记录测试者卡在哪、翻回哪、想跳过哪。
4. 测试者自认"能开始用了" → **记 T1**。
5. 做一个 3 分钟理解检查（§4）：通过 → 记 T2；不通过 → 继续读，记下还缺什么。

## 4. 理解检查（3 题，验"真懂"还是"错觉懂"）

1. 用自己的话说出 **Spec / Prompt / Loop / Workflow 各管什么**？
2. 要起一个新项目，**最少**先定哪几样（对应最低强制版）？
3. **HG-2 什么时候触发**？（探边界：改 Spec 才触发、Loop 内调参不触发——这正是 OPEN-1 的盲区，看新人能否自己嗅到）

## 5. 判据（开测前冻结，勿事后改）

- **T1 > 2 小时** → 可执行性**不合格**（目标用户是无 onboarding 的个人）。
- **误解点 ≥ 2 个落在核心**（七层职责 / Spec↔Prompt 边界 / HG 触发）→ 核心表达不清，**重写那几节**。
- 理解检查答对 **< 2/3** → 没真懂，T1 是错觉，不算数。
- **三态裁定**：**PASS**（T1≤2h 且核心无误解且检查≥2/3）/ **改进**（部分超标，列具体节）/ **重做**（T1 远超 或 核心多处误解）。
- **推广限定**（v1.6.3 修订）：若仅测了 A 档测试者，结论须标注"经有 AI 协作经验者验证，不可推广至新手或非同构读者场景"。若测了 A + B + C 三档，结论可标注覆盖范围。此限定防止用"有经验者读得懂"论证"所有人都读得懂"——即防止 OPEN-4 成为循环论证。

## 6. 输出 + 写回

- 一页记录：T1、T2、**误解点清单**（章节 + 原话 + 他以为的意思）、**经验前置声明**、放弃信号、三态裁定、**推广范围标注**（仅测了哪些档位 → 结论适用于哪些人群）。
- 写回框架 §1.6 OPEN-4 状态；若"改进 / 重做" → 起一次 Retrospect。

## 成本

1 个测试者、1 次、半天内完成。**这是当前性价比最高的待决项**——成本极低、信息量极大。
