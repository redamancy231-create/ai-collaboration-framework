# 可调参数索引

> **版本**: v1.0（初始创建）  
> **日期**: 2026-06-21  
> **生成模型**: DeepSeek-V4-Pro (via Claude Code CLI shell)  
> **来源**: v1.6.3 维护流程补全——Qwen qwen3.7-max 完备性审查（2026-06-21）指出"配置参数散落各层定义中，读者无法快速定位哪些数字是可调的"。  
> **定位**: 配套目录外挂——参考工具，非强制合规项。将框架中显式或隐式的"魔法数字"集中为一表，供读者和未来维护者快速定位。

## 使用说明

- **可调性**：硬约束 = 违反会触发死亡判据或安全规则，不可直接调整；可调 = 阈值可基于经验修改但需记录理由；待校准 = 当前值为初始设定，需要试跑数据校准
- **调整流程**：硬约束的修改走 HG-2（Spec 变更闸门）；可调参数的修改走 Dev Log 记录 + 影响评估；待校准参数经试跑数据验证后更新
- 本表不替代各章节中的参数说明——它是指向各章节的索引，使用前应核实源章节中的值是否与本表一致

## 维护说明

- 新增参数时同步更新本表
- 参数值变更时更新"当前值"列并在 Dev Log 中记录理由
- 至少每两个 Minor 版本做一次全表同步检查（与源章节交叉验证数值一致性）
- "待校准"参数优先级最高——它们是框架已知的盲区

## 已知局限

- 本表为手动维护，可能存在与源章节的漂移
- "可调 vs 硬约束"的边界判定基于编辑者判断，部分条目可能有灰色地带
- 部分参数的"当前值"散落在叙述性文本中而非显式声明——提取时可能有疏漏
- 不覆盖源代码脚本中的硬编码参数（如 Python 脚本中的路径/超时/迭代次数）
