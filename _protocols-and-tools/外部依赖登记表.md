> **版本**: v1.0（初始创建）

> **生成模型**: DeepSeek-V4-Pro (via Claude Code CLI shell)  
> **来源**: v1.6.3 维护流程补全——框架受众与稳定性分析的两路异后端独立审查（Codex GPT-5.5 魔鬼代言人 + Qwen qwen3.7-max 完备性检查，2026-06-21）共同识别的结构性缺口。两后端在"外部依赖建模缺失"上零分歧收敛。

## 目的

框架重度依赖 Claude Code CLI 的工具能力边界和特定 AI 模型的行为特征。工具链和模型是 AI 协作领域变化最快的部分。本登记表提供当前依赖的快照，帮助在以下场景快速评估影响范围：
- 任一依赖出现 breaking change 公告时
- 框架 Minor 升级前
- 新增一个外部依赖时

## 检查节奏

- **每次 Minor 升级前**：全量检查工具链 + 模型 + 平台依赖
- **任一依赖出现 breaking change 公告时**：立即检查该依赖所在行
- **每季度（或 ≥5 个版本后）**：全量检查外部方法论参考状态
- **Patch 升级**：仅当改动涉及受影响的章节时才检查对应依赖

## 影响评估模板

当某依赖发生变化时，填写以下模板：

```
- 依赖 ID: ____
- 变化描述: ____

- 修订内容: ____
- 执行日期: ____
- 执行者: ____
```

## 已知局限

- 本登记表为快照工具，不提供自动监测——依赖变化靠人工检查节奏发现
- 当前仅覆盖框架直接操作中使用的依赖，未穷举所有潜在依赖
- "外部方法论参考"的"变化信号"定义模糊——方法论范式转移的判断标准待细化
- 本表初次创建（v1.0），尚未经过完整检查周期的实证验证
