# 任务：为 Kimi-K2.7-Code 设计 prompt 偏置审查提示词

## 背景

在《AI协作项目全生命周期框架》项目中，有一个"协议 2：OPEN-1 verify 降级替代"。该协议的核心逻辑是：

1. 框架的 OPEN-1 问题 = "离散人审测不出连续 AI 方向漂移"
2. 作者（DeepSeek-v4-pro）认为这是高严重性结构问题
3. 但没有零卷入人类专家验证过
4. 降级替代方案：把这个框架文档喂给一个独立后端（计划用 Qwen3.7-Max），用一份"冷读 prompt"让它找结构性缺口——但不提 OPEN-1 这个词
5. 如果审查者独立命中了 OPEN-1 同构问题 → OPEN-1 获独立佐证
6. 如果没命中 → OPEN-1 严重性降级

**这个冷读 prompt 已经被写好了，如下**：

```
你是一位方法论审阅者。请审查以下这份方法论文档的结构性缺口。

【审查对象】
AI协作项目全生命周期框架.md（随附）

【审查任务】
1. 找出这份文档的结构性缺口——不是措辞问题、不是错别字，是"设计层面的致命问题"。
2. 按杀伤力排序，列出你认为最严重的 3 个。
3. 每个缺口说明：是什么、为什么严重、作者是否已经意识到（看文档是否提到过）。

【审查约束】
- 只基于这份文档本身判断，不引入外部知识。
- 如果某个缺口作者已经在文档里承认了（如 §1.6 的待决项），标注"作者已承认"，但仍可评估其严重性是否被低估。
- 不要客气，不要给好话。

【输出格式】
缺口1：[标题]
  - 是什么：
  - 为什么严重：
  - 作者是否已意识到：
  - 严重性评分（1-10）：
（缺口2、3 同结构）
```

## 你的任务

你（ChatGPT-5.5）之前审查 GLM-5.2 编辑时，对这个冷读 prompt 提出了担忧：**它可能隐含方向性引导**。

现在需要你设计一份审查提示词，交给 **Kimi-K2.7-Code（Kimi Code CLI）** 执行。Kimi 的任务是：**专门审查上述冷读 prompt 是否含有方向性偏置，是否会污染冷读的独立性**。

## 设计要求

1. Kimi 的审查任务应该只针对 prompt 本身，不需要看框架文档
2. 审查重点：这个 prompt 的措辞是否会**系统性地**引导审查者走向某个特定方向（OPEN-1 的方向），而非让审查者自然发现
3. 如果有偏置——哪里有问题、严重程度、怎么修改
4. 如果没有偏置——为什么它通过了偏置检查
5. 给 Kimi 的提示词应该包含明确的审查维度和判据

## 输出

请直接输出给 Kimi-K2.7-Code 的完整审查提示词（中文）。格式应该可以直接保存为文件、通过 `cat prompt | kimi exec` 或类似方式喂给 Kimi。
